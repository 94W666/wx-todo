# DEV_RULES.md 项目开发强制规则
> 本文档为健身打卡小程序开发唯一强制执行标准，Claude所有操作必须100%遵循本规则，无任何例外。违反规则的输出视为无效，需立即修正。

---

## 一、核心提速铁则：单次需求最小闭环原则
1.  任何开发需求，严格限定在用户指定的**单个模块/单个功能/单个文件**内，绝对禁止超出需求范围的开发、修改、重构。
2.  单次需求仅完成最小可用闭环，禁止提前开发未提及的后续功能、禁止过度设计、禁止重复造轮子。
3.  必须基于`CLAUDE.md`中已确认的项目进度和代码开发，绝对不修改、不覆盖、不重构已正常运行的历史代码，仅新增/修改本次需求指定的内容。

## 二、需求执行标准流程（零方向返工）
接到开发需求后，必须严格按以下顺序执行，禁止跳过任何步骤：
1.  **范围锁定**：先读取`CLAUDE.md`明确项目进度、技术栈、已完成功能，再复述本次需求的核心目标、修改范围、涉及文件，用户确认无误后再进入下一步。
2.  **方案确认**：仅输出本次开发的「实现思路、涉及文件、核心修改点」，不输出完整代码，用户确认方案可行后，再进入代码开发环节。
3.  **代码开发**：严格按确认的方案开发，仅修改指定文件，全程遵循本规范的代码要求。
4.  **进度同步**：开发完成后，自动按规则更新`CLAUDE.md`进度文档，无需用户额外指令。
5.  **标准交付**：按本规范固定格式交付内容，无冗余信息。

## 三、代码开发强制规范
1.  **命名规范**：所有变量、方法、页面、文件采用小驼峰命名，禁止拼音、无意义单字母命名，命名必须见名知意。
2.  **注释规范**：
    - 每个js文件顶部必须加文件功能注释，每个方法必须加功能、入参、出参注释；
    - 核心逻辑分支必须加行内注释，禁止无注释代码；
    - 注释统一使用中文，简洁清晰，无冗余废话。
3.  **文件修改规则**：
    - 仅修改本次需求指定的文件，新增文件必须符合微信小程序项目目录规范；
    - 修改已有文件时，仅改动本次需求相关的代码块，保留原有正常运行的代码，禁止全量重写文件；
    - 禁止修改`project.config.json`等项目核心配置文件，除非用户明确要求。
4.  **代码质量要求**：
    - 所有数据操作、异步方法必须加异常捕获，有明确的兜底逻辑；
    - 代码必须符合微信小程序官方开发规范，可直接复制到项目中运行，无语法报错；
    - 禁止引入需求中未提及的第三方插件、框架、依赖。

## 四、输出范围红线禁令（解决token超限、无效内容）
1.  禁止一次性输出超过2个文件的完整代码，复杂模块必须分文件、分步骤输出，用户确认上一个文件无误后，再输出下一个。
2.  禁止输出与本次需求无关的解释、科普、额外方案，仅输出用户要求的核心内容。
3.  禁止重复输出已在`CLAUDE.md`、本规则文档中明确的内容，禁止反复复述需求。
4.  禁止输出大段测试数据、示例内容，仅提供核心代码和必要的使用说明。
5.  禁止擅自新增需求中未提及的功能、UI、交互，所有超出需求的内容必须先经用户确认。

## 五、进度自动同步规则
每次开发需求完成后，必须自动执行以下操作，无需用户额外发送指令：
1.  读取项目根目录的`CLAUDE.md`，严格遵循其固定模板结构；
2.  全量更新「当前项目全量进度总结」板块，清晰说明项目整体开发进度；
3.  在「已完成功能明细」板块，按固定格式新增本次完成的功能内容；
4.  同步更新「项目最新文件结构树」，确保和本地项目文件100%一致；
5.  更新「待开发任务清单」，标记已完成任务，调整后续开发优先级；
6.  直接写入本地`CLAUDE.md`文件，完成后仅给用户输出3条核心更新摘要，禁止输出完整文档内容。

## 六、异常处理强制规则
1.  开发过程中，若发现已有代码与本次需求冲突、依赖缺失、技术不兼容等问题，必须立即停止开发，先给用户输出「问题说明+2个可落地的解决方案」，用户确认方案后再继续执行，禁止擅自修改已有代码解决问题。
2.  若用户需求不明确、有歧义，必须先追问明确核心目标和边界，禁止凭猜测开发，避免返工。
3.  若开发过程中发现需求无法实现，必须第一时间说明原因，给出替代方案，禁止输出无法运行的无效代码。

## 七、交付物固定格式
每次开发完成后，必须严格按以下格式交付，禁止缺项：
 
 
本次开发完成情况
 
1. 核心目标：【本次需求完成的核心功能】
2. 涉及文件：【本次修改/新增的所有文件完整路径】
3. 核心修改点：【3条以内的核心实现说明】
4. 运行说明：【代码如何使用、如何验证效果】
5. 下一步建议：【基于本次开发，后续的开发优先级建议】
 
 
 
对应代码文件
 
【仅输出本次修改/新增的文件代码，单个文件单独输出，标注完整文件路径】
  

## 八、微信小程序专属适配规则
1.  页面开发必须遵循微信小程序「json+js+wxml+wxss」四文件结构，新增页面必须同步在`app.json`的`pages`数组中注册，禁止遗漏。
2.  数据存储必须使用微信小程序官方`wx.storage`API，禁止使用浏览器端不兼容的存储API。
3.  所有交互、UI必须适配微信小程序基础库2.0以上版本，禁止使用已废弃的API。
4.  样式开发必须使用rpx单位适配不同屏幕，禁止使用固定px单位写死尺寸。
