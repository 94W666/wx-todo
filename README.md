# 健身打卡小程序 - Fitness Check-in WeChat Mini Program

![WeChat Mini Program](https://img.shields.io/badge/WeChat-Mini_Program-green)
![Version](https://img.shields.io/badge/version-v1.3.1-blue)
![License](https://img.shields.io/badge/license-MIT-yellow)
![Base Library](https://img.shields.io/badge/base_library-2.19.0-lightgrey)

一个功能完整的健身打卡微信小程序，集游戏化段位系统、打卡热力图、智能训练计划于一体，帮助用户科学规划训练并保持运动习惯。

**App ID:** `wx2f1da9f8bec7ce8e`

## ✨ 核心特性

### 🏆 游戏化段位系统
- **8级段位体系**：从"青铜战士"到"不朽神话"的成长之路
- **动态升级**：基于连续打卡天数自动计算段位
- **视觉反馈**：每个段位有专属颜色和称号
- **升级提示**：显示距离下一段位所需天数

### 📊 打卡热力图
- **月历视图**：直观展示每月打卡情况
- **智能配色**：绿色表示已打卡，灰色表示未打卡
- **月份切换**：支持左右箭头浏览历史记录
- **日期识别**：准确显示星期和日期布局

### 📈 智能训练管理
- **双训练计划**：五分化训练(胸→背→肩→臂→腿) / 三分化训练(推→拉→腿)
- **智能部位识别**：基于昨日打卡自动计算今日训练部位
- **明日预览**：打卡后显示明日训练计划

### 📱 用户数据统计
- **总打卡天数**：累计训练次数统计
- **部位训练次数**：各身体部位训练频次分析
- **月度统计**：当月训练数据汇总
- **训练频率**：周/月训练规律分析
- **计划执行率**：训练计划完成度统计

### 😴 睡眠记录
- **睡眠时长记录**：5个时长选项(5小时及以下→9小时及以上)
- **睡眠体感记录**：4个体感选项(神清气爽→噩梦连连)
- **历史记录查看**：按日期展示睡眠质量
- **数据持久化**：本地存储所有睡眠记录

### 🔄 连续打卡统计
- **智能计算**：准确计算连续打卡天数，支持跨月跨年
- **双重统计**：当前连续天数 + 历史最高记录
- **防重复打卡**：一日内多次点击只记录一次

## 🚀 快速开始

### 前提条件
1. 安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)
2. 注册微信小程序账号并获取 AppID
3. 本项目使用基础库版本 2.19.0（支持懒加载代码注入）

### 安装步骤
1. **克隆项目**
   ```bash
   git clone https://github.com/94W666/wx-todo.git
   cd fitness-checkin
   ```

2. **导入项目到微信开发者工具**
   - 打开微信开发者工具
   - 选择"导入项目"
   - 选择本项目目录
   - 输入你的 AppID（或使用测试号）
   - 点击"导入"

3. **运行项目**
   - 导入后项目将自动编译
   - 在模拟器中查看小程序效果
   - 可通过"预览"功能在手机端体验

## 📖 使用指南

### 首次使用
1. 打开小程序，首页显示"青铜战士"段位
2. 热力图显示当月日历，所有日期为灰色（未打卡）
3. 点击"点击打卡"按钮完成首次打卡
4. 观察段位变化和热力图今日变绿

### 核心功能操作
- **每日打卡**：首页点击"点击打卡"记录今日训练
- **切换计划**：进入"计划"页面选择五分化/三分化训练
- **记录睡眠**：进入"睡眠记录"页面选择时长和体感
- **查看历史**：点击热力图左右箭头查看不同月份记录

### 训练逻辑
- **智能循环**：系统基于昨日打卡自动计算今日训练部位
- **计划切换**：切换计划后，训练部位循环相应调整
- **连续记录**：连续打卡天数决定段位等级
- **数据持久**：所有记录本地存储，重启小程序不丢失

## 🏗️ 技术架构

### 技术栈
- **框架**：微信小程序原生框架
- **语言**：JavaScript + WXML + WXSS
- **存储**：微信官方 Storage API
- **基础库**：2.19.0（支持懒加载代码注入）
- **适配**：rpx 单位实现多屏幕适配

### 核心模块
```javascript
// 主要功能模块分布
- pages/index/      # 首页：段位系统、热力图、打卡功能
- pages/plan/       # 训练计划管理
- pages/sleep/      # 睡眠记录管理
- utils/util.js     # 核心工具函数
```

### 性能优化
- **懒加载代码注入**：`"lazyCodeLoading": "requiredComponents"`
- **按需注入**：减少小程序启动时间和内存占用
- **数据缓存**：合理使用本地存储提升响应速度

## 🛠️ 开发指南

### 开发环境配置
1. 确保微信开发者工具版本支持基础库 2.19.0+
2. 开启 ES6 转 ES5、增强编译等选项
3. 配置项目设置中的 `lazyCodeLoading` 为 `"requiredComponents"`

### 代码规范
- **命名规范**：小驼峰命名法，见名知意
- **注释要求**：所有方法需有中文注释说明功能、参数、返回值
- **文件结构**：遵循微信小程序四文件结构（js+json+wxml+wxss）
- **样式单位**：使用 rpx 适配不同屏幕，禁用固定 px

### 核心工具函数说明
```javascript
// utils/util.js 主要函数
- calculateStreakDays()    # 计算连续打卡天数（支持跨月跨年）
- calculateRank()          # 计算用户段位（8级系统）
- generateHeatmapData()    # 生成热力图月历数据
- getCurrentPart()         # 智能识别当前训练部位
```

### 添加新功能
1. 在 `CLAUDE.md` 的待办清单中添加任务描述
2. 按微信小程序规范创建新页面或组件
3. 在 `app.json` 中注册新页面（如添加页面）
4. 遵循开发规则文档 `DEV_RULES.md` 的要求
5. 开发完成后更新 `CLAUDE.md` 进度文档
